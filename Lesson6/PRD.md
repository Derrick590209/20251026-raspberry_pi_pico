# 產品需求文件 (PRD) - MQTT 數據監控儀表板

## 專案概述

本專案旨在建立一個基於 Streamlit 的 Web 應用程式，作為 MQTT 協議的訂閱者（Subscriber），用於即時監控和顯示客廳環境數據。

## 專案目標

1. 建立一個 Streamlit Web 應用程式
2. 實作 MQTT 訂閱者功能，接收來自 MQTT Broker 的即時數據
3. 提供直觀的用戶介面，顯示環境監控資訊
4. 自動儲存接收到的數據，以便後續分析

## 用戶介面需求

應用程式需要顯示以下資訊：

1. **電燈狀態顯示**
   - 顯示電燈目前的開關狀態（開/關）

2. **客廳溫度顯示**
   - 即時顯示當前客廳溫度數值

3. **客廳濕度顯示**
   - 即時顯示當前客廳濕度數值

4. **溫濕度趨勢圖表**
   - 使用圖表形式展示溫度和濕度的歷史變化趨勢
   - X 軸：日期和時間
   - Y 軸：溫度/濕度數值

## 功能規格

### 1. 數據顯示功能
- 應用程式主要功能為**顯示**，不包含控制功能
- 所有顯示的數據需即時更新，反映最新的 MQTT 訊息

### 2. 數據儲存功能
- 訂閱者接收到的所有數據需自動儲存為 Excel 檔案格式
- 儲存的數據應包含時間戳記，以便追蹤數據的接收時間
- Excel 檔案應包含完整的歷史記錄，方便後續數據分析

## 技術要求

- **框架**: Streamlit
- **通訊協議**: MQTT (使用 paho-mqtt 客戶端)
- **數據格式**: Excel 檔案（使用 pandas 或 openpyxl 等套件）
- **圖表庫**: Streamlit 內建圖表功能或 matplotlib/plotly

## 非功能需求

- 應用程式需能穩定運行，持續監聽 MQTT 訊息
- 數據儲存功能需可靠，確保數據不會遺失
- 用戶介面需簡潔易讀，提供良好的使用者體驗
